🔗 DSU (Disjoint Set Union)

Union-Find বিচ্ছিন্ন সেট ডেটা স্ট্রাকচার

সংক্ষিপ্ত পরিচিতি: DSU বা Disjoint Set Union হলো একটি ডেটা স্ট্রাকচার যা একাধিক সেট (set) কে দক্ষতার সাথে ম্যানেজ করে। এটি মূলত দুটি অপারেশন সাপোর্ট করে: Find (কোনো element কোন সেটে আছে তা খোঁজা) এবং Union (দুটি সেটকে একত্রিত করা)।

📌 কখন DSU ব্যবহার করবেন?

DSU তখনই ব্যবহার করবেন যখন সমস্যাটি নিচের বৈশিষ্ট্যগুলোর একটি বা একাধিক দেখায়:

  • 🔹 Connectivity / Connected Components: দুটি node বা vertex আসলে connected কিনা তা চেক করতে হবে।
  • 🔹 Dynamic Connectivity: edge যোগ বা বিয়োগ করার পরেও connectivity বজায় রাখতে হবে।
  • 🔹 Grouping / Clustering: একাধিক element কে একটি গ্রুপে বা কম্পোনেন্টে বিভক্ত করতে হবে।
  • 🔹 Cycle Detection: গ্রাফে cycle আছে কিনা তা খুঁজে বের করতে হবে (বিশেষ করে Undirected Graph-এ)।
  • 🔹 Equivalence Relations: "সমতুল্য" (equivalent) সম্পর্ক নির্ধারণ করতে হবে।

🎯 এটি কি টাইপের অ্যালগোরিদম?

উত্তর: DSU একটি Data Structure + Algorithm উভয়ই। এটি মূলত একটি Set-based / Graph-based অ্যালগোরিদম ক্যাটাগরিতে পড়ে।

এটি সাধারণত নিচের ধরনের সমস্যায় ব্যবহৃত হয়:

  • Graph Problems: Connected components, MST (Kruskal's Algorithm)
  • Network Problems: Friend circles, social networks
  • Grid Problems: Island counting, region merging
  • Dynamic Problems: Online queries for connectivity

💡 Interview-এ কী দেখে বুঝবেন এটি DSU প্রব্লেম?

Interview-এ নিচের কিওয়ার্ড বা ক্লু গুলো শুনলে বুঝবেন এটি DSU সল্ভ করা যাবে:

  • ✅ "Connected" / "Connection" / "Connectivity" - যদি বলে দুটি জিনিস connected কিনা চেক করতে হবে।
  • ✅ "Group" / "Cluster" / "Component" - যদি element গুলোকে গ্রুপে ভাগ করতে হয়।
  • ✅ "Merge" / "Union" / "Combine" - যদি বারবার কিছু মার্জ করতে হয়।
  • ✅ "Cycle" / "Redundant" / "Loop" - যদি cycle বা redundant edge খুঁজতে হয়।
  • ✅ "Network" / "Graph" / "Edges" - যদি graph বা network টাইপের ডেটা থাকে।
  • ✅ "Equivalent" / "Same group" / "Belongs to" - যদি সমতুল্য সম্পর্ক নির্ধারণ করতে হয়।
  • ✅ "Dynamic" / "Online queries" - যদি বারবার query আসে connectivity এর জন্য।

🛠️ DSU Implementation (Python)

নিচে একটি standard DSU implementation দেওয়া হলো (Path Compression + Union by Rank সহ):

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.count = n  # number of connected components

    def find(self, x):
        # Path compression
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        # Union by rank
        root_x = self.find(x)
        root_y = self.find(y)
        if root_x == root_y:
            return False  # already connected
        if self.rank[root_x] < self.rank[root_y]:
            self.parent[root_x] = root_y
        elif self.rank[root_x] > self.rank[root_y]:
            self.parent[root_y] = root_x
        else:
            self.parent[root_y] = root_x
            self.rank[root_x] += 1
        self.count -= 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)

⏱️ Time & Space Complexity

  • Time Complexity: Find এবং Union অপারেশনের জন্য O(α(n)) যেখানে α হলো Inverse Ackermann Function (practically constant time)।
  • Space Complexity: O(n) - parent এবং rank array এর জন্য।
